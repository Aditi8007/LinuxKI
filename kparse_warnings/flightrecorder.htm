LinuxKI Warning

 

Flightrecorder software impacting benchmark system by as much as 33%

Date: 09/30/2013 

Problem

HP was performing a benchmark with a DL980 G7 server  (80 cores, HT disable) running RHEL 6.4 connected to two 3PAR 7450 arrays with all new solid-state disks (SSD).   The goal of the benchmark was to achieve over 1 million I/O per second (IOPS) using 4K random reads.     The system was only achieving about 750K IOPS instead of the target 1 million I/O.

Investigation

The Linux KI Toolset was used to collect a KI trace sample for 20 seconds.    The CPU/RUNQ Report (kiinfo -kirunq) showed very high System CPU utilization on the CPUs where the I/O benchmark tool was running:

The debugfs filesystem was not explicitly mounted, however, after mounting the debugfs manually, we found that ftrace was indeed enabled.   Examining the /etc/sysconfig revealed the following file:

$ cat trace-cmd.config

# ftrace based flight recorder configuration file.

 

# trace-cmd options

OPTS="-b 2048 -e block -e irq -e mce -e module -e power -e sched -e signal -e sys_enter -e sys_exit -e timer -e workqueue"

The trace-cmd was configured as part of the flightrecorder software package:

$ rpm --query flightrecorder

flightrecorder-0.91-1.el6.x86_64

The flight-recorder software trace-cmd (ftrace front-end) to collect trace data to the circular trace ring buffers.   The ring buffers can then be analyzed in the event of a system crash to help identify the root cause.  

However, in the heavy I/O workload, the trace events logged (specifically the irq and block requests) were adding considerable overhead.    The flight-recorder software was stopped using trace-cmd service as follows:

$ service trace-cmd stop

Once the flight-record was stopped, the I/O performance benchmark successfully completed over 1 million IOPS, a 33% increase in I/O throughput.

Conclusion

The flightrecorder software can cause significant performance degradation in some workloads, especially high I/O throughput workload and should be disabled or the software package should be removed.

Based on this investigation, the runki script for the Linux KI Toolset was modified to print the following warning if flightrecorder is installed and running on the system:

!!! WARNING flightrecorder is installed and active

!!! WARNING Please note that flightrecorder can impact system performance

!!! You can temporarily disable the flight recorder using the following command

# service trace-cmd stop
